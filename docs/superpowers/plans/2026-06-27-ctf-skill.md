# CTF Skill Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an "ultimate" CTF skill — a triage-and-execute brain that classifies a challenge, loads the matching playbook, and drives it autonomous-first by delegating tool execution to the separately-maintained kali skill.

**Architecture:** Approach A — single invocable skill with progressive disclosure. A lean `SKILL.md` routes (Triage → Dispatch → Capture) and per-category depth lives in `references/*.md` loaded on demand. The skill names tools + args; the kali skill owns connection/execution. A dependency-free bash lint plus a triage rubric provide verification.

**Tech Stack:** Markdown (skill + references), bash (lint script via the Bash tool), the existing kali skill for execution. No language runtime dependency beyond bash + coreutils (grep).

**Shared reference shape** — every `references/*.md` MUST contain these five H2 headers, in order:

```
## Triggers
## Workflow
## Commands
## Pitfalls
## Flag extraction
```

- **Triggers** — fast signals that route a challenge here (file types, keywords, port/service hints).
- **Workflow** — ordered steps from first look to flag.
- **Commands** — concrete tool + args to run *via the kali skill*. NEVER hardcode host/auth/ssh — write the tool invocation only (e.g. `nmap -sC -sV -p- <target>`), prefixed by the convention "Run via the kali skill:".
- **Pitfalls** — common dead-ends and gotchas.
- **Flag extraction** — how the flag typically appears for this category and how to grab/validate it.

```bash
#!/usr/bin/env bash
# Verifies the CTF skill follows its required structure.
# Exit 0 = pass, 1 = fail. Dependency-free (bash + grep).
set -u
cd "$(dirname "$0")/.." || exit 1
fail=0
err() { echo "FAIL: $*"; fail=1; }

# --- SKILL.md ---
[ -f SKILL.md ] || err "SKILL.md missing"
if [ -f SKILL.md ]; then
  head -1 SKILL.md | grep -q '^---$' || err "SKILL.md missing frontmatter opener"
  grep -q '^name:' SKILL.md || err "SKILL.md frontmatter missing name:"
  grep -q '^description:' SKILL.md || err "SKILL.md frontmatter missing description:"
  for s in Triage Dispatch Capture Safety; do
    grep -qi "## .*$s" SKILL.md || err "SKILL.md missing '## $s' section"
  done
fi

# --- references ---
REQ=("## Triggers" "## Workflow" "## Commands" "## Pitfalls" "## Flag extraction")
EXPECTED=(web crypto forensics stego reversing pwn misc-osint network)
for name in "${EXPECTED[@]}"; do
  f="references/$name.md"
  [ -f "$f" ] || { err "$f missing"; continue; }
  for h in "${REQ[@]}"; do
    grep -qF "$h" "$f" || err "$f missing section '$h'"
  done
  # commands must not hardcode the kali host/transport
  if grep -qiE 'ssh +[a-z]+@|192\.168\.|nohup .*2>&1 &' "$f"; then
    err "$f hardcodes kali connection details (delegate to kali skill instead)"
  fi
done

# --- assets + rubric ---
[ -f assets/notes-template.md ] || err "assets/notes-template.md missing"
[ -f tests/triage-rubric.md ] || err "tests/triage-rubric.md missing"

if [ "$fail" -eq 0 ]; then echo "PASS: ctf skill lint"; fi
exit "$fail"
```

````markdown
# CTF

The brain for CTF events. Triage a challenge, load the right playbook, drive it to a
flag. Tool execution is delegated to the **kali skill** — this skill names the tool and
arguments to run; it never hardcodes the host, auth, or transport.

## How to use this skill

Run the loop: **Triage → Dispatch → Capture**. Stay autonomous-first — run
recon/exploit steps without asking; checkpoint only before destructive or irreversible
actions, and confirm authorization before scanning/attacking any target.

## Triage

Classify the challenge into one category using fast signals. Gather signals first
(via the kali skill where execution is needed):

- Artifact on disk: run `file`, `strings -n 8`, and `binwalk` on it.
- Challenge title and prompt wording (titles often hint the category).
- A URL or web service → web. A bare host/IP or "get root/user.txt" → network.

Routing table (pick the best match; note your confidence):

| Signals | Category | Reference |
|---------|----------|-----------|
| URL, web app, HTTP service, login form, "admin", cookies | web | `references/web.md` |
| ciphertext, keys, "encrypted", base64/hex blobs, RSA params | crypto | `references/crypto.md` |
| disk image, pcap, memory dump, office/pdf docs, carving | forensics | `references/forensics.md` |
| image/audio with hidden data, "look closer", LSB | stego | `references/stego.md` |
| ELF/PE/Mach-O to understand, "what's the password", license check | reversing | `references/reversing.md` |
| ELF/PE with a network service, overflow, "nc host port", shellcode | pwn | `references/pwn.md` |
| odd encodings, trivia, OSINT, a found hash, "google it" | misc-osint | `references/misc-osint.md` |
| target host/box, "get a shell", privesc, user.txt/root.txt | network | `references/network.md` |

If signals are ambiguous, pick the highest-confidence category and proceed; re-triage
on dead-end.

## Dispatch

1. Read the chosen `references/<category>.md` and follow its Workflow.
2. Seed a workspace: copy `assets/notes-template.md` to `ctf-notes/<challenge>/notes.md`.
3. Execute the category's Commands through the kali skill. Log each command + key
   output to the notes file.

## Capture

1. Extract the candidate flag.
2. Validate against the flag format. Default patterns: `flag{...}`, `CTF{...}`,
   `picoCTF{...}`, and any event-specific format the user provides — ask for the format
   if unknown and a candidate doesn't match.
3. Record the flag + the exact reproduction steps in the notes file, then report it.

If stuck: run the chosen reference's Pitfalls/fallback list, then re-triage to the next
most likely category before escalating to the user.

## Safety

- Confirm scope/authorization before scanning or attacking any target.
- Checkpoint before destructive or irreversible actions.
- CTF targets only unless the user explicitly authorizes a pentest engagement.
- Keep all findings in `ctf-notes/`.
````

### Task 4: references/web.md

- **Triggers:** URL/web app, HTTP service, login form, cookies/JWT, `?id=` params, admin panel, CMS.
- **Workflow:** enumerate → identify stack → test each injection class → exploit → loot flag (filesystem, DB, admin area).
- **Commands (concrete):**
  - Recon: `gobuster dir -u <url> -w /usr/share/wordlists/dirb/common.txt -x php,txt,html`; `nikto -h <url>`; check `<url>/robots.txt`, `/sitemap.xml`, response headers.
  - SQLi: `sqlmap -u "<url>?id=1" --batch --dbs` then `--dump`.
  - NoSQLi: auth bypass with operators `$ne`, `$regex`, `$where` (JSON body or `param[$ne]=`).
  - XSS: reflected/stored payload `<script>...</script>`; note for CTF it's usually to steal an admin cookie.
  - Command injection: `; id`, `| id`, `$(id)`, backticks in params.
  - SSTI: probe `{{7*7}}` / `${7*7}`; escalate per engine (Jinja2 `{{config}}`, etc.).
  - Directory traversal: `<url>?file=../../../../etc/passwd`.
  - File upload: bypass filters (double extension, magic bytes, `.phtml`), then browse to the upload for RCE.
  - WordPress: `wpscan --url <url> --enumerate u,vp`; brute with `wpscan ... --passwords`.
- **Pitfalls:** missing trailing slash on gobuster; WAF/rate limits; flag in DB vs. on disk; cookie `httpOnly`; client-side-only checks (read JS).
- **Flag extraction:** grep dumped DB / read files for the flag regex; check `/flag`, `/flag.txt`, env vars, admin dashboard.

### Task 5: references/crypto.md

- **Triggers:** ciphertext blobs, base64/hex, "encrypted", RSA params (n,e,c), repeated-key hints, a found hash.
- **Workflow:** identify the scheme → apply the matching attack → decode → flag.
- **Commands/techniques (concrete):**
  - Identify: paste into CyberChef ("Magic"); check base64 (`base64 -d`), hex (`xxd -r -p`).
  - Classical: Caesar/ROT (try all 25 shifts); Vigenère (frequency analysis / online Guballa); substitution (quipqiup).
  - XOR / one-time pad: single-byte XOR brute; detect key reuse (crib-drag) when a pad is reused.
  - RSA: small-`e` cube root; shared/weak modulus → factor with FactorDB or `sympy.factorint`; Wiener for small `d`; use `RsaCtfTool`.
  - Hashing: identify with `hashid <hash>`; crack `john --format=<fmt> --wordlist=/usr/share/wordlists/rockyou.txt <hashfile>` or `hashcat -m <mode> <hashfile> rockyou.txt`; or CrackStation lookup.
- **Pitfalls:** confusing encoding (reversible) with encryption; wrong hash mode; rockyou still gzipped (`gunzip` first); endianness in XOR.
- **Flag extraction:** decoded plaintext usually IS the flag or contains it; pipe through the flag regex.

### Task 6: references/forensics.md

- **Triggers:** disk image (`.img`,`.dd`,`.E01`), pcap (`.pcap(ng)`), memory dump (`.raw`,`.mem`), office/pdf docs, "recover", "carve".
- **Workflow:** identify container → extract/carve → analyze the right layer → flag.
- **Commands (concrete):**
  - Identify/extract: `file <file>`; `binwalk -e <file>`; `foremost -i <file> -o out/`; `photorec` for carving.
  - Metadata: `exiftool <file>`; `strings -n 8 <file>`.
  - PCAP: open in Wireshark; `tshark -r <file> -Y http.request`; follow TCP/HTTP streams; extract files (File→Export Objects); USB-keystroke recovery via `tshark -r <file> -Y 'usb.capdata'` then map HID codes.
  - Memory: `volatility3 -f <file> windows.info` then `pslist`, `cmdline`, `filescan`, `dumpfiles`.
  - NTFS ADS: list/read alternate data streams on a mounted image.
  - PDF: `qpdf --qdf --object-streams=disable <in> <out>`; `pdf-parser`; search layers (Inkscape) and strings.
- **Pitfalls:** truncated/wrong-magic files; needing the right volatility profile/symbols; flag split across packets; hidden in slack space.
- **Flag extraction:** `strings <artifact> | grep -iE 'flag\{|CTF\{'`; reconstruct exfiltrated file then grep.

### Task 7: references/stego.md

- **Triggers:** image/audio file with a too-large size, "look closer", "hidden", "spectrogram", visually plain PNG/WAV.
- **Workflow:** metadata → embedded files → bit-plane/channel → format-specific tool → flag.
- **Commands (concrete):**
  - First pass: `exiftool <file>`; `strings -n 8 <file>`; `binwalk -e <file>`.
  - Images: `zsteg -a <png/bmp>`; `steghide extract -sf <jpg/wav>` (try empty + guessed passphrase); `stegseek <file> /usr/share/wordlists/rockyou.txt`; channel/level tweaks in StegSolve or `convert` (`-separate`, contrast).
  - Audio: open spectrogram in Audacity/Sonic Visualiser; check for DTMF/Morse; `strings`.
  - Text: detect whitespace stego (tabs/spaces) — decode with a whitespace/snow tool.
- **Pitfalls:** assuming one tool covers all formats; passphrase required for steghide; flag in spectrogram not bytes; double-embedded layers.
- **Flag extraction:** extracted payload/visual text IS the flag; pipe extracted bytes through the flag regex.

### Task 8: references/reversing.md

- **Triggers:** an executable to understand (ELF/PE/Mach-O), "find the password", license/key check, no network service.
- **Workflow:** static triage → identify protection/packing → static or dynamic analysis → recover logic/flag.
- **Commands (concrete):**
  - Triage: `file <bin>`; `strings -n 6 <bin>`; check packers (UPX → `upx -d`).
  - Static: `objdump -d -M intel <bin>`; load in Ghidra (decompile `main`); `nm`, `readelf -a`.
  - Dynamic: `gdb <bin>` with pwndbg/gef — `break main`, `run`, inspect comparisons; `ltrace`/`strace <bin>` to watch libc/string calls (`strcmp` reveals passwords).
  - Anti-debug: watch for `ptrace`/`isDebuggerPresent`/`int 3`; patch or set return values in gdb.
- **Pitfalls:** stripped binaries (no symbols); password compared after transform (read the algo); statically-linked noise; wrong architecture in gdb.
- **Flag extraction:** recovered constant/derived string IS the flag; or run the binary with the correct input and read its output.

### Task 9: references/pwn.md

- **Triggers:** binary + a remote service (`nc <host> <port>`), source hints at overflow/format string, "shell", provided libc.
- **Workflow:** checksec → find the bug → craft input → leak/bypass mitigations → get shell/flag.
- **Commands (concrete):**
  - Recon: `checksec --file=<bin>`; `file <bin>`; identify libc version.
  - Offset: cyclic pattern — `pwn cyclic 200`, crash, `pwn cyclic -l <value>`.
  - Classic BOF: overwrite return to `win()`/`system("/bin/sh")`.
  - Format string: leak stack/`printf` to read or overwrite (`%p`,`%n`); GOT overwrite.
  - ROP: build chain with ROPgadget/pwntools when NX is on; ret2libc using a leaked libc address (defeat ASLR via leak).
  - Tooling: pwntools template — `from pwn import *; p = remote('<host>',<port>)` (or `process('./<bin>')`), build payload, `p.interactive()`.
  - Mitigations cheat: NX → ROP/ret2libc; canary → leak it first; PIE/ASLR → leak an address; partial RELRO → GOT overwrite possible.
- **Pitfalls:** libc mismatch (use the provided one + `LD_PRELOAD`); bad-char bytes; stack alignment (movaps) for `system`; off-by-one on offset.
- **Flag extraction:** drop to shell then `cat flag.txt`; or the service prints the flag after exploitation.

### Task 10: references/misc-osint.md

- **Triggers:** trivia, weird encodings/encoding chains, OSINT (find a person/place/account), a bare hash/zip/archive, "google it".
- **Workflow:** identify the puzzle type → decode/search/crack → flag.
- **Commands/techniques (concrete):**
  - Encodings: chain-decode in CyberChef (base64/32, hex, URL, ROT, morse, brainfuck); `file` unknown blobs.
  - OSINT: reverse image search; EXIF GPS from photos (`exiftool`); pivot on usernames/emails; Wayback Machine; certificate/`whois`/DNS records.
  - Archives/protected files: convert then crack — `zip2john <f.zip> > h`, `ssh2john <key> > h`, `7z2john`, then `john --wordlist=/usr/share/wordlists/rockyou.txt h`.
  - GTFOBins lookup for any "abuse this binary" misc task.
- **Pitfalls:** rabbit holes (verify the title hint); over-thinking low-point challenges; rockyou gzipped; assuming encryption when it's just encoding.
- **Flag extraction:** decoded/cracked output IS the flag; validate against the flag regex.

### Task 11: references/network.md

- **Triggers:** a target host/IP/box, "get a shell", "user.txt/root.txt", privesc, lateral movement.
- **Workflow:** recon → enumerate services → exploit a foothold → stabilize shell → privesc → loot → (lateral movement).
- **Commands (concrete):**
  - Recon: `nmap -sn <range>` (host discovery); `nmap -sC -sV -p- -oN scan.txt <target>`; `nmap -sU --top-ports 50 <target>`.
  - Enumerate by service: web → `gobuster`/`nikto`; SMB → `enum4linux -a <target>`, `smbmap -H <target>`, `netexec smb <target>`; DNS → `dig axfr @<target> <domain>`; brute → `hydra -l <user> -P /usr/share/wordlists/rockyou.txt <target> <service>`.
  - Exploit: `searchsploit <service version>`; Metasploit `use <module>` → set RHOSTS/LHOST → run.
  - Shell stabilization: `python3 -c 'import pty;pty.spawn("/bin/bash")'`, then `Ctrl-Z`, `stty raw -echo; fg`, `export TERM=xterm`.
  - Reverse shells/listener: `nc -lvnp <port>` listener; bash/python reverse-shell one-liners to `<lhost>:<port>`.
  - Privesc: run `linpeas.sh`/`winpeas`; check `sudo -l` + GTFOBins; SUID (`find / -perm -4000`); cron via `pspy`; kernel exploit suggester.
  - File transfer: `python3 -m http.server` + `wget`/`curl`; `certutil -urlcache -f` / PowerShell `IEX` on Windows.
- **Pitfalls:** tcpwrapped → slow scan (`-T2`, no aggressive flags); unstable shells; LHOST wrong on multi-NIC; flag is `user.txt` then `root.txt`.
- **Flag extraction:** `cat user.txt`/`root.txt`; `grep -rIE 'flag\{|CTF\{' / 2>/dev/null` once you have access.
